    Home

    Operaton is a community-driven, free, and open-source BPMN engine.

    It inherited its source code from the legacy of Camunda, Activiti and their predecessors. Camunda 7 was an important contribution to the open-source BPMN community. Building on this legacy, we aim to create a thriving future for a truly free and open-source BPMN engine.

    Our Promise

    • Operaton is a state-of-the-art BPMN engine
    • Forever free and open source under the Apache 2.0 license
    • Community-owned by a legal foundation (currently in progress)
    • The trademark Operaton will be owned by the instituted legal foundation

    Features & Goals
    • Easy migration from Camunda 7 (see Documentation and OpenRewrite scripts)
    • Up-to-date dependencies (e.g. increased Java and Spring base versions)
    • Focus on security and stability
    • Completely new web apps, rewritten with accessibility and performance in mind
    • Improved documentation (see Documentation)
    • An inclusive and welcoming community (join the Forum and Chat (Slack))

    Principles
    Community ownership
    Operaton shall always be a community-owned free open-source project (legal work in progress). There shall be no private entity which is able to legally claim its ownership or divert the goals of the community. See our Code of Conduct to learn how we foster an open, welcoming, diverse, inclusive, and healthy community.
    Quality
    It shall be held to high-quality standards regarding every aspect (e.g. see our SonarQube activtities and our GitHub issues).
    Transparency
    Operaton is developed in the open. All decisions are transparently made by its community for its community. (See our regular communication channels).
    Self ownership
    There shall be no hidden dependencies making it hard to build it on your own. Everything is easy to run by yourself (see Getting Started Guide (currently WIP))

    Roadmap

    We want to keep Operaton users informed and know what to expect next and when. This also shall show where the current priorities of the contributers lie. If you have other priorities you may share them in our forum or contribute changes yourself (with the help of the community).

    Release 1.0 – ASAP

    • Ensure stability and security
    • Drop-in replacement for existing web apps
    • Improve tests
    • Update dependencies

    New and improved web apps – 2025

    • Migrate away from legacy AngularJS
    • Improve user experience based on feedback from the community for casual and power users
    • Include accessibility as a core principle
    • Include all important features without restriction (like paid licenses)
    • Implement important features requested by the community

    Further Development

    • No breaking changes
    • New features based on user needs
    • Modular approach using extensions to keep the core small and maintainable
    • Enhance maintainablity and therefore security by further cleaning up the code base
    • Improve documentation by adding example projects and tutorials
