Operaton is a community driven open-source BPMN engine inhereting its source from the legacy of Camunda, Activiti and their predecessors. We believe that the latest open version of Camunda 7 is an important contribution for the open source BPMN community, which should be kept secure and accessible.

  • Free and open source: Apache 2.0
  • Community owned project (including trademarks WIP)
  • Accessible, fast and intuitive web apps

    Features and Goals
    • Operaton inherits all features from Camunda 7 and introduces no ciritcaly breaking changes
    • We focus on security and stability by increasing the Java and Spring base version, as well as making sure that dependencies are using current releases
    • We work on better performance and clean up the code base for better maintainability. This ensures that security issues or general bugs are resolved fast
    • We integrate essential features (like oauth), which are desired by the users
    • We improve documentation for a better developer experience
    • We rewrite the web applications and introduce an accessible and secure web frontend. The UX shall integrate the needs of casual users, as well as power users
    • Operaton shall be an inclusive community

    Principles
    • Operaton shall be a community owned project (legal work in pogress)
    • It shall be held to high quality standards regarding every aspect
    • There shall be no hidden dependencies making it hard to build it on your own. Everything is easy to run yourself
    • There shall be no private entity which is able to legally claim its ownership

    Roadmap

    We want to keep Operatons users informed and know what to expect next and when. This also shall show where the current priorities of the contributers lie. If you have other priorities you may share them in our forum or contribute changes yourself (with the help of the community).

    • Release 1.0 – asap
      • Ensure stability and security
      • Drop-in replacement for existing web apps
      • Improve tests
      • Update dependencies
    • New and improved web apps – 2025
      • Migrate away from legacy AngluarJS
      • Improve user experience based on feedback from the community for casual and power users
      • Include accessibility as core principle
      • Include all important features without restriction (like paid licenses)
      • Implement imporant features wanted by the community
    • Further Development
      • No breaking changes
      • New features based on user needs
      • Modular approach using extensions to keep the core small and maintainable
      • Enhance maintainablity and therefore security by further cleaning up the code base
      • Improve documentation, add example projects and tutorials
